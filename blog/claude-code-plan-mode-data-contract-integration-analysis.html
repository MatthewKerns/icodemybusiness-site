Data Contract & Integration Analysis: How Three AI Systems Talk to Each Other

Matthew Kerns March 12, 2026 12 min read

What you're reading: This is a real Claude Code plan mode output. I asked Claude to examine the data contract between three of my systems, check for latest updates, and map out how everything integrates. This is the kind of analysis that would take a senior engineer hours — Claude mapped it out in minutes.

The Problem

I run multiple interconnected systems: an AI agent layer (Agent-OS), a company operations platform (InfinityVault-OS), and a client delivery system (iCodeMyBusiness). I needed to understand exactly how data flows between them — what's the contract, what are the boundaries, and where are the integration points.

I opened Claude Code plan mode and asked it to examine the data contract between Agent-OS and InfinityVault-OS, check for the latest updates, and explore the internal operations data model. Here's what came back.

Key Findings

1. Agent-OS Data Contract with InfinityVault-OS

Contract Location: packages/agent-os-contract/

Architecture: Agent-OS serves as an AI agent layer that integrates with two business systems:

  • InfinityVault-OS: Company operations management (EOS framework)
  • iCodeMyBusiness: Client work and project delivery

Data Contract Structure:

  • Types-only package with zero dependencies
  • Clear API endpoints defined for both systems
  • Domain boundary enforcement (each system owns specific data)

2. Data Models Being Exchanged

EOS Management Data (InfinityVault-OS ↔ Agent-OS)

Core Types:

  • Rocks: 90-day company priorities with status tracking
  • Issues: Company problems requiring resolution
  • Tasks: Employee tasks derived from rocks/issues
  • Plans: Daily/weekly/quarterly planning documents
  • Vision: Company strategic direction
  • Seats: Organizational roles and responsibilities

Key Endpoints:

Method Endpoint Purpose
GET/PUT /api/eos/rocks Rock management
GET/PUT /api/eos/issues Issue tracking
GET/POST/PUT /api/eos/tasks Task operations
GET/POST /api/eos/plans Planning documents
GET /api/eos/vision Company vision (read-only)

iCodeMyBusiness Data (ICMB ↔ Agent-OS)

Core Types:

  • Clients: Customer relationships with health tracking
  • Projects: Client work with revenue and strategic value
  • Deliverables: Specific outputs with deadlines
  • Communications: Client interaction tracking
  • Business Health: Aggregated metrics and alerts

Key Endpoints:

Method Endpoint Purpose
GET /api/icmb/clients Client management
GET/PUT /api/icmb/projects Project tracking
GET /api/icmb/deliverables Deliverable status
GET /api/icmb/health Business metrics

3. Latest Agent-OS Updates

Multi-Company Briefing System

  • New MultiCompanyBriefingAggregator fetches data from both InfinityVault and iCodeMyBusiness
  • LLM-powered NarrativeBriefingGenerator creates persuasive daily briefings
  • BriefingConversationLoop enables email-based briefing adjustments
  • Protection for unpaid projects from being dropped (growth strategy)

Email Reply Processing

  • Parses user replies to daily briefings using Claude LLM
  • Extracts rock/issue/todo status updates
  • Applies updates back to source systems
  • Supports bidirectional email conversations

4. iCodeMyBusiness Internal Operations

Three Kanban Board System

  1. Lead Generation Funnel: Awareness → Engaged → Call → Proposal
  2. Discovery & Planning: Onboarding → Process Mapping → Design → Specs
  3. Implementation & Delivery: Sprint → Progress → Testing → Review → Delivered

Comprehensive Data Model

  • 8 major table groups tracking leads, clients, projects, SOPs, content
  • Integration points for Unipile, Stripe, Fathom, Calendly, Loom
  • Dashboard implementation at /dashboard/ with 10 operational views

Arise Group AI Integration

  • Referenced as active work: "Agent Team Integration"
  • Building parallel AI agent system
  • This appears to be the connection to Agent-OS ecosystem

  1. Check API Endpoints: Test the EOS and ICMB API endpoints to ensure they match the contract definitions
  2. Review Type Definitions: Verify that the types in @agent-os/contract match actual API responses

  4. Validate Email Processing: Send test replies to daily briefings and verify status updates are applied
  5. Monitor Integration Points: Check logs for any contract violations or type mismatches
